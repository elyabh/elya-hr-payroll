# Cursor Rules for ELYA HR & Payroll

Follow the ELYA architecture strictly.

## Development Rules

- Use Modular Monolith architecture.
- Keep Domain independent.
- Use Application layer for use cases.
- Use Infrastructure for external services.
- Use Persistence for database.
- Use API only for endpoints.
- Always enforce TenantId.
- Always enforce permissions.
- Do not generate unrelated code.
- Work only on the current sprint.
